epub-audiobook

A Python CLI that converts an EPUB into a single audiobook MP3 with navigable chapter markers, using Google Cloud Text-to-Speech Neural2 voices.

PythonGoogle Cloud TTSSSMLffmpegResumable

Chapter navigation works in Apple Books, VLC, and most audiobook players via ID3 CHAP/CTOC frames — no need for the M4B container.

What makes it fast

Most EPUB→MP3 scripts re-encode every chunk through pydub, run sequentially, and emit tiny SSML payloads. This one does the opposite:

📦 Packed SSML

Packs SSML into ~4.8 KB chunks (Google's hard limit), sentence-aligned.

⚡ Parallel synthesis

Default 8 workers under a token-bucket rate limiter.

🎙️ No re-encode

Concatenates with ffmpeg -c copy — fast even on long books.

🔖 Chapter markers

Writes ID3 CHAP/CTOC frames for real chapter navigation.

↩️ Resumable

Per-chunk MP3s are cached in --workdir, keyed by chapter + index.

✍️ Emphasis-aware

Italics/bold map to SSML <emphasis> when the span stays within a sentence.

Requirements

  • Python 3.10+
  • ffmpeg on the PATH
  • A Google Cloud project with the Text-to-Speech API enabled

Python deps: ebooklib, beautifulsoup4, lxml, requests, tqdm, mutagen, google-auth.

Install

python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
brew install ffmpeg     # macOS; `apt install ffmpeg` on Linux

Authentication

Pick one. A service-account JSON is recommended; API keys travel in URLs.

# Option A: service account (recommended)
export GOOGLE_APPLICATION_CREDENTIALS=/path/to/sa.json

# Option B: API key
export GOOGLE_TTS_API_KEY=YOUR_KEY
Never commit credentials. The repo's .gitignore blocks *.json, *.key, and common credential filename patterns.

Usage

# Dry run — parse EPUB, report chapter/chunk/char counts, no API calls.
python audiobook.py book.epub out.mp3 --dry-run

# Real run
python audiobook.py book.epub out.mp3

# Different voice / speaking rate
python audiobook.py book.epub out.mp3 --voice en-US-Neural2-D --speaking-rate 1.05

# Resumable run with explicit working dir
python audiobook.py book.epub out.mp3 --workdir ./chunks --keep-workdir
Always --dry-run first to see the character count before spending any TTS quota.

How it works

Parse EPUB (ebooklib)→ Extract chapters (BeautifulSoup)→ Sentence-split + SSML chunk (~4.8KB)→ Synthesize in parallel (TTS)→ ffmpeg -c copy concat→ Write ID3 CHAP/CTOC

Each chunk is cached by chapter + index, so an interrupted run resumes without re-synthesizing what's already done.

Cost & free tier

Neural2 voices include 1,000,000 characters/month free, then $0.000016/char. A typical 300-page non-fiction book is ~500–700k characters — usually within the free tier. The --dry-run count tells you before you commit.

Troubleshooting

SymptomLikely cause / fix
Auth errorsSet GOOGLE_APPLICATION_CREDENTIALS or GOOGLE_TTS_API_KEY; confirm the TTS API is enabled.
No chapter markersPlayer may not read ID3 chapters — try VLC or Apple Books.
ffmpeg: not foundInstall ffmpeg and ensure it's on PATH.
Run interruptedRe-run with the same --workdir; cached chunks are reused.
